### <a name="strategies"></a><span data-ttu-id="32958-125">策略</span><span class="sxs-lookup"><span data-stu-id="32958-125">Strategies</span></span>

<span data-ttu-id="32958-126">可以使用多种备用策略在区域间实现分布式计算。</span><span class="sxs-lookup"><span data-stu-id="32958-126">Many alternative strategies are available for implementing distributed compute across regions.</span></span> <span data-ttu-id="32958-127">这些策略必须根据特定业务要求和应用程序的情况量身定制。</span><span class="sxs-lookup"><span data-stu-id="32958-127">These must be tailored to the specific business requirements and circumstances of the application.</span></span> <span data-ttu-id="32958-128">大体而言，方法可分为以下几类：</span><span class="sxs-lookup"><span data-stu-id="32958-128">At a high level, the approaches can be divided into the following categories:</span></span>

- <span data-ttu-id="32958-133">**热备份（主动/主动）**：应用程序设计为在多个区域中接收生产负载。</span><span class="sxs-lookup"><span data-stu-id="32958-133">**Hot Spare (Active/Active)**: The application is designed to receive production load in multiple regions.</span></span> <span data-ttu-id="32958-134">为每个区域中云服务所配置的容量可能高于灾难恢复用途所需的容量。</span><span class="sxs-lookup"><span data-stu-id="32958-134">The cloud services in each region might be configured for higher capacity than required for disaster recovery purposes.</span></span> <span data-ttu-id="32958-135">或者，云服务可在灾难或故障转移时进行扩展。</span><span class="sxs-lookup"><span data-stu-id="32958-135">Alternatively, the cloud services might scale out as necessary at the time of a disaster and failover.</span></span> <span data-ttu-id="32958-136">这种方法需要在应用程序设计中进行大量的投资，但是也具有明显的好处。</span><span class="sxs-lookup"><span data-stu-id="32958-136">This approach requires substantial investment in application design, but it has significant benefits.</span></span> <span data-ttu-id="32958-137">这些好处包括恢复时间更短且更有保证、连续测试所有恢复位置和对容量的有效使用。</span><span class="sxs-lookup"><span data-stu-id="32958-137">These include low and guaranteed recovery time, continuous testing of all recovery locations, and efficient usage of capacity.</span></span>

<span data-ttu-id="32958-138">关于分布式设计的完整讨论不属于本文档的范畴。</span><span class="sxs-lookup"><span data-stu-id="32958-138">A complete discussion of distributed design is outside the scope of this document.</span></span> <span data-ttu-id="32958-139">有关更多信息，请参阅 [Azure 应用程序的灾难恢复和高可用性](https://aka.ms/drtechguide)。</span><span class="sxs-lookup"><span data-stu-id="32958-139">For further information, see [Disaster Recovery and High Availability for Azure Applications](https://aka.ms/drtechguide).</span></span>

# <a name="failure-mode-analysis"></a><span data-ttu-id="a9492-103">故障模式分析</span><span class="sxs-lookup"><span data-stu-id="a9492-103">Failure mode analysis</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="a9492-104">故障模式分析 (FMA) 是一个过程，用于通过标识系统中可能的故障点将复原能力内置到系统中。</span><span class="sxs-lookup"><span data-stu-id="a9492-104">Failure mode analysis (FMA) is a process for building resiliency into a system, by identifying possible failure points in the system.</span></span> <span data-ttu-id="a9492-105">FMA 应该是体系结构和设计阶段的一部分，以便从一开始就将故障恢复能力内置到系统中。</span><span class="sxs-lookup"><span data-stu-id="a9492-105">The FMA should be part of the architecture and design phases, so that you can build failure recovery into the system from the beginning.</span></span>

<span data-ttu-id="a9492-106">下面是执行 FMA 的常规过程：</span><span class="sxs-lookup"><span data-stu-id="a9492-106">Here is the general process to conduct an FMA:</span></span>

1. <span data-ttu-id="a9492-107">标识系统中的所有组件。</span><span class="sxs-lookup"><span data-stu-id="a9492-107">Identify all of the components in the system.</span></span> <span data-ttu-id="a9492-108">包括外部依赖项，比如标识提供者、第三方服务等等。</span><span class="sxs-lookup"><span data-stu-id="a9492-108">Include external dependencies, such as as identity providers, third-party services, and so on.</span></span>
2. <span data-ttu-id="a9492-109">对于每个组件，标识可能发生的潜在故障。</span><span class="sxs-lookup"><span data-stu-id="a9492-109">For each component, identify potential failures that could occur.</span></span> <span data-ttu-id="a9492-110">单个组件可能具有多种故障模式。</span><span class="sxs-lookup"><span data-stu-id="a9492-110">A single component may have more than one failure mode.</span></span> <span data-ttu-id="a9492-111">例如，应分开考虑读取故障和写入故障，因为其影响和可行的缓解措施有所不同。</span><span class="sxs-lookup"><span data-stu-id="a9492-111">For example, you should consider read failures and write failures separately, because the impact and possible mitigations will be different.</span></span>
3. <span data-ttu-id="a9492-112">根据总体风险对每种故障模式评分。</span><span class="sxs-lookup"><span data-stu-id="a9492-112">Rate each failure mode according to its overall risk.</span></span> <span data-ttu-id="a9492-113">请考虑以下因素：</span><span class="sxs-lookup"><span data-stu-id="a9492-113">Consider these factors:</span></span>

   - <span data-ttu-id="a9492-118">对应用程序的可用性、数据丢失、货币成本和业务中断有什么影响？</span><span class="sxs-lookup"><span data-stu-id="a9492-118">What is the impact on the application, in terms of availability, data loss, monetary cost, and business disruption?</span></span>

4. <span data-ttu-id="a9492-119">对于每种故障模式，确定应用程序将如何响应和恢复。</span><span class="sxs-lookup"><span data-stu-id="a9492-119">For each failure mode, determine how the application will respond and recover.</span></span> <span data-ttu-id="a9492-120">考虑在成本与应用程序复杂程度之间作出折衷。</span><span class="sxs-lookup"><span data-stu-id="a9492-120">Consider tradeoffs in cost and application complexity.</span></span>

<span data-ttu-id="a9492-121">作为 FMA 过程的起点，本文包含了潜在故障模式及其缓解措施的目录。</span><span class="sxs-lookup"><span data-stu-id="a9492-121">As a starting point for your FMA process, this article contains a catalog of potential failure modes and their mitigations.</span></span> <span data-ttu-id="a9492-122">该目录按技术或 Azure 服务分类，另外还有一个针对应用程序级设计的常规目录。</span><span class="sxs-lookup"><span data-stu-id="a9492-122">The catalog is organized by technology or Azure service, plus a general category for application-level design.</span></span> <span data-ttu-id="a9492-123">该目录并不详尽，但涵盖许多核心 Azure 服务。</span><span class="sxs-lookup"><span data-stu-id="a9492-123">The catalog is not exhaustive, but covers many of the core Azure services.</span></span>

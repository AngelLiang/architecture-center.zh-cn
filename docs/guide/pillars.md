## <a name="scalability"></a><span data-ttu-id="23131-117">可伸缩性</span><span class="sxs-lookup"><span data-stu-id="23131-117">Scalability</span></span>

<span data-ttu-id="23131-118">可伸缩性是指系统处理增加的负载的能力。</span><span class="sxs-lookup"><span data-stu-id="23131-118">Scalability is the ability of a system to handle increased load.</span></span> <span data-ttu-id="23131-119">应用程序可通过两种主要方式进行扩展。</span><span class="sxs-lookup"><span data-stu-id="23131-119">There are two main ways that an application can scale.</span></span> <span data-ttu-id="23131-120">垂直扩展（*纵向*扩展）指增加资源的容量，例如通过使用更大的 VM。</span><span class="sxs-lookup"><span data-stu-id="23131-120">Vertical scaling (scaling *up*) means increasing the capacity of a resource, for example by using a larger VM size.</span></span> <span data-ttu-id="23131-121">水平扩展（*横向*扩展）指添加资源的新实例，比如 VM 或数据库副本。</span><span class="sxs-lookup"><span data-stu-id="23131-121">Horizontal scaling (scaling *out*) is adding new instances of a resource, such as VMs or database replicas.</span></span>

<span data-ttu-id="23131-122">水平扩展相较垂直扩展具有明显优势：</span><span class="sxs-lookup"><span data-stu-id="23131-122">Horizontal scaling has significant advantages over vertical scaling:</span></span>

- <span data-ttu-id="23131-123">真正的云规模。</span><span class="sxs-lookup"><span data-stu-id="23131-123">True cloud scale.</span></span> <span data-ttu-id="23131-124">可将应用程序设计为在数百个甚至数千个节点上运行，其规模是在单个节点上无法达到的。</span><span class="sxs-lookup"><span data-stu-id="23131-124">Applications can be designed to run on hundreds or even thousands of nodes, reaching scales that are not possible on a single node.</span></span>
- <span data-ttu-id="23131-125">水平扩展具有弹性。</span><span class="sxs-lookup"><span data-stu-id="23131-125">Horizontal scale is elastic.</span></span> <span data-ttu-id="23131-126">如果负载增加，可以添加更多实例；在较安静的时间段，则可以删除实例。</span><span class="sxs-lookup"><span data-stu-id="23131-126">You can add more instances if load increases, or remove them during quieter periods.</span></span>
- <span data-ttu-id="23131-127">可以按计划或为响应负载变化，自动触发横向扩展。</span><span class="sxs-lookup"><span data-stu-id="23131-127">Scaling out can be triggered automatically, either on a schedule or in response to changes in load.</span></span>
- <span data-ttu-id="23131-128">横向扩展可能比纵向扩展更便宜。</span><span class="sxs-lookup"><span data-stu-id="23131-128">Scaling out may be cheaper than scaling up.</span></span> <span data-ttu-id="23131-129">运行多个小型 VM 比运行单个大型 VM 的成本更低。</span><span class="sxs-lookup"><span data-stu-id="23131-129">Running several small VMs can cost less than a single large VM.</span></span>
- <span data-ttu-id="23131-130">水平扩展还可通过添加冗余提高复原能力。</span><span class="sxs-lookup"><span data-stu-id="23131-130">Horizontal scaling can also improve resiliency, by adding redundancy.</span></span> <span data-ttu-id="23131-131">如果某个实例出现故障，应用程序将继续运行。</span><span class="sxs-lookup"><span data-stu-id="23131-131">If an instance goes down, the application keeps running.</span></span>

<span data-ttu-id="23131-132">垂直扩展的一个优点是，扩展时不必对应用程序进行任何更改。</span><span class="sxs-lookup"><span data-stu-id="23131-132">An advantage of vertical scaling is that you can do it without making any changes to the application.</span></span> <span data-ttu-id="23131-133">但会在某个时候达到极限，即，再也无法纵向扩展。</span><span class="sxs-lookup"><span data-stu-id="23131-133">But at some point you'll hit a limit, where you can't scale any up any more.</span></span> <span data-ttu-id="23131-134">这时，任何进一步的扩展都只能是水平扩展。</span><span class="sxs-lookup"><span data-stu-id="23131-134">At that point, any further scaling must be horizontal.</span></span>

<span data-ttu-id="23131-135">必须将水平扩展设计到系统中。</span><span class="sxs-lookup"><span data-stu-id="23131-135">Horizontal scale must be designed into the system.</span></span> <span data-ttu-id="23131-136">例如，可通过将 VM 放在负载均衡器后面来横向扩展 VM。</span><span class="sxs-lookup"><span data-stu-id="23131-136">For example, you can scale out VMs by placing them behind a load balancer.</span></span> <span data-ttu-id="23131-137">但池中的每个 VM 都必须能够处理任何客户端请求，因此应用程序必须无状态或将状态存储在外部（例如，在分布式缓存中）。</span><span class="sxs-lookup"><span data-stu-id="23131-137">But each VM in the pool must be able to handle any client request, so the application must be stateless or store state externally (say, in a distributed cache).</span></span> <span data-ttu-id="23131-138">托管 PaaS 服务通常内置水平扩展和自动扩展。</span><span class="sxs-lookup"><span data-stu-id="23131-138">Managed PaaS services often have horizontal scaling and auto-scaling built in.</span></span> <span data-ttu-id="23131-139">能轻松扩展这些服务是使用 PaaS 服务的主要优点。</span><span class="sxs-lookup"><span data-stu-id="23131-139">The ease of scaling these services is a major advantage of using PaaS services.</span></span>

<span data-ttu-id="23131-140">不过，只添加更多实例并不意味着就扩展了应用程序。</span><span class="sxs-lookup"><span data-stu-id="23131-140">Just adding more instances doesn't mean an application will scale, however.</span></span> <span data-ttu-id="23131-141">它可能只是将瓶颈推到了其他地方。</span><span class="sxs-lookup"><span data-stu-id="23131-141">It might simply push the bottleneck somewhere else.</span></span> <span data-ttu-id="23131-142">例如，如果扩展 Web 前端以处理更多客户端请求，则可能在数据库中触发锁争用。</span><span class="sxs-lookup"><span data-stu-id="23131-142">For example, if you scale a web front-end to handle more client requests, that might trigger lock contentions in the database.</span></span> <span data-ttu-id="23131-143">然后，你就得考虑其他对策，比如乐观并发或数据分区，以提高数据库的吞吐量。</span><span class="sxs-lookup"><span data-stu-id="23131-143">You would then need to consider additional measures, such as optimistic concurrency or data partitioning, to enable more throughput to the database.</span></span>

<span data-ttu-id="23131-144">始终执行性能和负载测试以发现这些潜在瓶颈。</span><span class="sxs-lookup"><span data-stu-id="23131-144">Always conduct performance and load testing to find these potential bottlenecks.</span></span> <span data-ttu-id="23131-145">系统的有状态部分（如数据库）是导致瓶颈最常见的原因，因此在设计水平扩展时需谨慎。</span><span class="sxs-lookup"><span data-stu-id="23131-145">The stateful parts of a system, such as databases, are the most common cause of bottlenecks, and require careful design to scale horizontally.</span></span> <span data-ttu-id="23131-146">解决一个瓶颈可能会暴露其他位置的其他瓶颈。</span><span class="sxs-lookup"><span data-stu-id="23131-146">Resolving one bottleneck may reveal other bottlenecks elsewhere.</span></span>

<span data-ttu-id="23131-147">使用[可伸缩性清单][scalability-checklist]从可伸缩性角度审查你的设计。</span><span class="sxs-lookup"><span data-stu-id="23131-147">Use the [Scalability checklist][scalability-checklist] to review your design from a scalability standpoint.</span></span>

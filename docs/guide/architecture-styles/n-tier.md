# <a name="n-tier-architecture-style"></a><span data-ttu-id="724ba-103">N 层体系结构样式</span><span class="sxs-lookup"><span data-stu-id="724ba-103">N-tier architecture style</span></span>

![N 层体系结构样式的逻辑图](./images/n-tier-logical.svg)

<span data-ttu-id="724ba-106">层是分离职责和管理依赖关系的方式。</span><span class="sxs-lookup"><span data-stu-id="724ba-106">Layers are a way to separate responsibilities and manage dependencies.</span></span> <span data-ttu-id="724ba-107">每个层都有特定的责任。</span><span class="sxs-lookup"><span data-stu-id="724ba-107">Each layer has a specific responsibility.</span></span> <span data-ttu-id="724ba-108">较高层可使用较低层中的服务，反之则不行。</span><span class="sxs-lookup"><span data-stu-id="724ba-108">A higher layer can use services in a lower layer, but not the other way around.</span></span>

<span data-ttu-id="724ba-109">层在物理上是分隔开的，在不同的计算机上运行。</span><span class="sxs-lookup"><span data-stu-id="724ba-109">Tiers are physically separated, running on separate machines.</span></span> <span data-ttu-id="724ba-110">一个层可直接调用另一个层，或使用异步消息传递（消息队列）。</span><span class="sxs-lookup"><span data-stu-id="724ba-110">A tier can call to another tier directly, or use asynchronous messaging (message queue).</span></span> <span data-ttu-id="724ba-111">虽然每个层可能托管在自己的层中，但这并不是必需的。</span><span class="sxs-lookup"><span data-stu-id="724ba-111">Although each layer might be hosted in its own tier, that's not required.</span></span> <span data-ttu-id="724ba-112">多个层可能托管在同一层上。</span><span class="sxs-lookup"><span data-stu-id="724ba-112">Several layers might be hosted on the same tier.</span></span> <span data-ttu-id="724ba-113">在物理上分隔层可以提高可伸缩性和复原能力，但因额外的网络通信也增加了延迟。</span><span class="sxs-lookup"><span data-stu-id="724ba-113">Physically separating the tiers improves scalability and resiliency, but also adds latency from the additional network communication.</span></span>

<span data-ttu-id="724ba-114">传统的三层应用程序有表示层、中间层和数据库层。</span><span class="sxs-lookup"><span data-stu-id="724ba-114">A traditional three-tier application has a presentation tier, a middle tier, and a database tier.</span></span> <span data-ttu-id="724ba-115">中间层是可选的。</span><span class="sxs-lookup"><span data-stu-id="724ba-115">The middle tier is optional.</span></span> <span data-ttu-id="724ba-116">更复杂的应用程序可以多于三层。</span><span class="sxs-lookup"><span data-stu-id="724ba-116">More complex applications can have more than three tiers.</span></span> <span data-ttu-id="724ba-117">上图显示具有两个中间层，且封装了不同功能区域的应用程序。</span><span class="sxs-lookup"><span data-stu-id="724ba-117">The diagram above shows an application with two middle tiers, encapsulating different areas of functionality.</span></span>

<span data-ttu-id="724ba-118">N 层应用程序可以有封闭的层体系结构或开放的层体系结构：</span><span class="sxs-lookup"><span data-stu-id="724ba-118">An N-tier application can have a **closed layer architecture** or an **open layer architecture**:</span></span>

- <span data-ttu-id="724ba-119">在封闭的层体系结构中，层只能调用紧邻的下一层。</span><span class="sxs-lookup"><span data-stu-id="724ba-119">In a closed layer architecture, a layer can only call the next layer immediately down.</span></span>

<span data-ttu-id="724ba-121">封闭的层体系结构限制层之间的依赖关系。</span><span class="sxs-lookup"><span data-stu-id="724ba-121">A closed layer architecture limits the dependencies between layers.</span></span> <span data-ttu-id="724ba-122">但是，如果一个层仅将请求传递到下一层，可能会产生不必要的流量。</span><span class="sxs-lookup"><span data-stu-id="724ba-122">However, it might create unnecessary network traffic, if one layer simply passes requests along to the next layer.</span></span>

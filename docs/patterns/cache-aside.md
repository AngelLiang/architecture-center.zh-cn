# <a name="cache-aside-pattern"></a><span data-ttu-id="cf9ac-104">缓存端模式</span><span class="sxs-lookup"><span data-stu-id="cf9ac-104">Cache-Aside pattern</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="cf9ac-105">将数据按需从数据存储加载到缓存中。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-105">Load data on demand into a cache from a data store.</span></span> <span data-ttu-id="cf9ac-106">这可提升性能，并且有助于在缓存中保存的数据与基础数据存储中的数据之间保持一致性。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-106">This can improve performance and also helps to maintain consistency between data held in the cache and data in the underlying data store.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="cf9ac-107">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="cf9ac-107">Context and problem</span></span>

<span data-ttu-id="cf9ac-108">应用程序使用缓存来改善对数据存储中保存的信息的重复访问。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-108">Applications use a cache to improve repeated access to information held in a data store.</span></span> <span data-ttu-id="cf9ac-109">但是，期望缓存的数据始终与数据存储中的数据完全一致是不切实际的。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-109">However, it's impractical to expect that cached data will always be completely consistent with the data in the data store.</span></span> <span data-ttu-id="cf9ac-110">应用程序应实施策略，帮助尽可能确保缓存中的数据为最新状态，并且检测和处理当缓存中的数据过期时所出现的情况。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-110">Applications should implement a strategy that helps to ensure that the data in the cache is as up-to-date as possible, but can also detect and handle situations that arise when the data in the cache has become stale.</span></span>

## <a name="solution"></a><span data-ttu-id="cf9ac-111">解决方案</span><span class="sxs-lookup"><span data-stu-id="cf9ac-111">Solution</span></span>

<span data-ttu-id="cf9ac-112">许多商业缓存系统提供直读和直写/后写操作。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-112">Many commercial caching systems provide read-through and write-through/write-behind operations.</span></span> <span data-ttu-id="cf9ac-113">在这些系统中，应用程序通过引用缓存来检索数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-113">In these systems, an application retrieves data by referencing the cache.</span></span> <span data-ttu-id="cf9ac-114">如果数据不在缓存中，则将从数据存储中检索数据并将其添加到缓存。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-114">If the data isn't in the cache, it's retrieved from the data store and added to the cache.</span></span> <span data-ttu-id="cf9ac-115">对缓存中保存的数据进行的任何修改还会自动写回到数据存储。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-115">Any modifications to data held in the cache are automatically written back to the data store as well.</span></span>

<span data-ttu-id="cf9ac-116">对于不提供此功能的缓存，使用缓存的应用程序将负责保存数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-116">For caches that don't provide this functionality, it's the responsibility of the applications that use the cache to maintain the data.</span></span>

<span data-ttu-id="cf9ac-117">通过实施缓存端策略，应用程序可以模拟直读缓存的功能。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-117">An application can emulate the functionality of read-through caching by implementing the cache-aside strategy.</span></span> <span data-ttu-id="cf9ac-118">此策略可按需将数据加载到缓存。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-118">This strategy loads data into the cache on demand.</span></span> <span data-ttu-id="cf9ac-119">下图演示使用缓存端模式在缓存中存储数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-119">The figure illustrates using the Cache-Aside pattern to store data in the cache.</span></span>

<span data-ttu-id="cf9ac-121">如果应用程序更新了信息，则可按照直写策略操作，方法是修改数据存储和使缓存中的相应项无效。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-121">If an application updates information, it can follow the write-through strategy by making the modification to the data store, and by invalidating the corresponding item in the cache.</span></span>

<span data-ttu-id="cf9ac-122">如果下一步需要该项，使用缓存端策略将导致可从数据存储检索更新后的数据，并将其添加回缓存。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-122">When the item is next required, using the cache-aside strategy will cause the updated data to be retrieved from the data store and added back into the cache.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="cf9ac-123">问题和注意事项</span><span class="sxs-lookup"><span data-stu-id="cf9ac-123">Issues and considerations</span></span>

<span data-ttu-id="cf9ac-124">在决定如何实现此模式时，请考虑以下几点：</span><span class="sxs-lookup"><span data-stu-id="cf9ac-124">Consider the following points when deciding how to implement this pattern:</span></span>

<span data-ttu-id="cf9ac-125">**已缓存数据的生存期**。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-125">**Lifetime of cached data**.</span></span> <span data-ttu-id="cf9ac-126">许多缓存实施过期策略，如果未在指定期间访问数据，则数据将失效并从缓存中删除。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-126">Many caches implement an expiration policy that invalidates data and removes it from the cache if it's not accessed for a specified period.</span></span> <span data-ttu-id="cf9ac-127">若要使缓存端有效，请确保过期策略与使用数据的应用程序的访问模式相匹配。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-127">For cache-aside to be effective, ensure that the expiration policy matches the pattern of access for applications that use the data.</span></span> <span data-ttu-id="cf9ac-128">过期期限不宜太短，因为可能导致应用程序不断从数据存储检索数据并将其添加到缓存。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-128">Don't make the expiration period too short because this can cause applications to continually retrieve data from the data store and add it to the cache.</span></span> <span data-ttu-id="cf9ac-129">同样，过期期限不宜太长，否则缓存数据可能会过期。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-129">Similarly, don't make the expiration period so long that the cached data is likely to become stale.</span></span> <span data-ttu-id="cf9ac-130">请记住，缓存最适用于相对静态的数据或经常读取的数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-130">Remember that caching is most effective for relatively static data, or data that is read frequently.</span></span>

<span data-ttu-id="cf9ac-131">**逐出数据**。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-131">**Evicting data**.</span></span> <span data-ttu-id="cf9ac-132">与数据起源的数据存储相比，大多数缓存的大小有限，并且在必要时逐出数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-132">Most caches have a limited size compared to the data store where the data originates, and they'll evict data if necessary.</span></span> <span data-ttu-id="cf9ac-133">大多数缓存采用最近最少使用策略来选择要逐出的项，但这也可自定义。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-133">Most caches adopt a least-recently-used policy for selecting items to evict, but this might be customizable.</span></span> <span data-ttu-id="cf9ac-134">配置缓存的全局过期属性和其他属性，以及每个已缓存项的过期属性，以确保缓存具有成本效益。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-134">Configure the global expiration property and other properties of the cache, and the expiration property of each cached item, to ensure that the cache is cost effective.</span></span> <span data-ttu-id="cf9ac-135">将全局逐出策略应用于缓存中的每个项并不总是适用。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-135">It isn't always appropriate to apply a global eviction policy to every item in the cache.</span></span> <span data-ttu-id="cf9ac-136">例如，如果从数据存储检索某个已缓存项的代价很高，则将该项保留在缓存中是有益的，尽管访问更频繁，但减少了成本高的项。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-136">For example, if a cached item is very expensive to retrieve from the data store, it can be beneficial to keep this item in the cache at the expense of more frequently accessed but less costly items.</span></span>

<span data-ttu-id="cf9ac-137">**填充缓存**。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-137">**Priming the cache**.</span></span> <span data-ttu-id="cf9ac-138">很多解决方案使用可能用作应用程序启动处理一部分的数据来预填充缓存。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-138">Many solutions prepopulate the cache with the data that an application is likely to need as part of the startup processing.</span></span> <span data-ttu-id="cf9ac-139">如果一些数据已过期或已逐出，则缓存端模式仍然十分有用。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-139">The Cache-Aside pattern can still be useful if some of this data expires or is evicted.</span></span>

<span data-ttu-id="cf9ac-140">**一致性**。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-140">**Consistency**.</span></span> <span data-ttu-id="cf9ac-141">实现缓存端模式并不能保证数据存储与缓存之间的一致性。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-141">Implementing the Cache-Aside pattern doesn't guarantee consistency between the data store and the cache.</span></span> <span data-ttu-id="cf9ac-142">外部进程可随时更改数据存储中的项，并且在下次加载该项之前，此更改不会反映在缓存中。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-142">An item in the data store can be changed at any time by an external process, and this change might not be reflected in the cache until the next time the item is loaded.</span></span> <span data-ttu-id="cf9ac-143">在跨数据存储复制数据的系统中，如果经常发生同步，则此问题可能会变得严重。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-143">In a system that replicates data across data stores, this problem can become serious if synchronization occurs frequently.</span></span>

<span data-ttu-id="cf9ac-144">**本地（内存中）缓存**。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-144">**Local (in-memory) caching**.</span></span> <span data-ttu-id="cf9ac-145">缓存可能是应用程序实例的本地缓存，并且存储在内存中。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-145">A cache could be local to an application instance and stored in-memory.</span></span> <span data-ttu-id="cf9ac-146">如果应用程序反复访问相同的数据，则缓存端在此环境中很有用。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-146">Cache-aside can be useful in this environment if an application repeatedly accesses the same data.</span></span> <span data-ttu-id="cf9ac-147">但是，因为本地缓存是专用的，因此不同的应用程序实例可各自拥有相同缓存数据的副本。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-147">However, a local cache is private and so different application instances could each have a copy of the same cached data.</span></span> <span data-ttu-id="cf9ac-148">缓存之间的此数据可快速变得不一致，因此有必要使专用缓存中保存的数据过期并更加频繁地刷新该数据。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-148">This data could quickly become inconsistent between caches, so it might be necessary to expire data held in a private cache and refresh it more frequently.</span></span> <span data-ttu-id="cf9ac-149">在这些情况下，请考虑研究共享或分布式缓存机制的使用。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-149">In these scenarios, consider investigating the use of a shared or a distributed caching mechanism.</span></span>

## <a name="example"></a><span data-ttu-id="cf9ac-161">示例</span><span class="sxs-lookup"><span data-stu-id="cf9ac-161">Example</span></span>

<span data-ttu-id="cf9ac-163">以下代码示例使用 [StackExchange.Redis](https://github.com/StackExchange/StackExchange.Redis) 客户端，这是针对 .NET 编写的 Redis 客户端库。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-163">This following code examples use the [StackExchange.Redis](https://github.com/StackExchange/StackExchange.Redis) client, which is a Redis client library written for .NET.</span></span> <span data-ttu-id="cf9ac-164">若要连接到 Azure Redis 缓存实例，请调用静态 `ConnectionMultiplexer.Connect` 方法并传入连接字符串。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-164">To connect to an Azure Redis Cache instance, call the static `ConnectionMultiplexer.Connect` method and pass in the connection string.</span></span> <span data-ttu-id="cf9ac-165">该方法返回表示连接的 `ConnectionMultiplexer`。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-165">The method returns a `ConnectionMultiplexer` that represents the connection.</span></span> <span data-ttu-id="cf9ac-166">共享应用程序中的 `ConnectionMultiplexer` 实例的一个方法是，拥有返回连接示例的静态属性（与下列示例类似）。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-166">One approach to sharing a `ConnectionMultiplexer` instance in your application is to have a static property that returns a connected instance, similar to the following example.</span></span> <span data-ttu-id="cf9ac-167">此方法是一种线程安全方法，仅初始化连接的一个实例。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-167">This approach provides a thread-safe way to initialize only a single connected instance.</span></span>

```csharp
private static ConnectionMultiplexer Connection;

// Redis Connection string info
private static Lazy<ConnectionMultiplexer> lazyConnection = new Lazy<ConnectionMultiplexer>(() =>

    string cacheConnection = ConfigurationManager.AppSettings["CacheConnection"].ToString();
    return ConnectionMultiplexer.Connect(cacheConnection);
```

<span data-ttu-id="cf9ac-168">以下代码示例中的 `GetMyEntityAsync` 方法演示如何实现缓存端模式。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-168">The `GetMyEntityAsync` method in the following code example shows an implementation of the Cache-Aside pattern.</span></span> <span data-ttu-id="cf9ac-169">此方法使用直读方法从缓存检索对象。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-169">This method retrieves an object from the cache using the read-through approach.</span></span>

<span data-ttu-id="cf9ac-170">通过将整数 ID 用作密钥来识别对象。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-170">An object is identified by using an integer ID as the key.</span></span> <span data-ttu-id="cf9ac-171">`GetMyEntityAsync` 方法尝试使用此密钥从缓存检索项。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-171">The `GetMyEntityAsync` method tries to retrieve an item with this key from the cache.</span></span> <span data-ttu-id="cf9ac-172">如果找到匹配项，则它将返回。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-172">If a matching item is found, it's returned.</span></span> <span data-ttu-id="cf9ac-173">如果缓存中没有匹配项，`GetMyEntityAsync` 方法将从数据存储检索对象、将其添加到缓存中，然后将其返回。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-173">If there's no match in the cache, the `GetMyEntityAsync` method retrieves the object from a data store, adds it to the cache, and then returns it.</span></span> <span data-ttu-id="cf9ac-174">从数据存储实际读取数据的代码取决于数据存储，所以它未在此处显示。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-174">The code that actually reads the data from the data store is not shown here, because it depends on the data store.</span></span> <span data-ttu-id="cf9ac-175">请注意，缓存的项已配置为过期，以防止在其他位置对其进行更新后它会变得陈旧。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-175">Note that the cached item is configured to expire to prevent it from becoming stale if it's updated elsewhere.</span></span>

```csharp
// Set five minute expiration as a default
private const double DefaultExpirationTimeInMinutes = 5.0;

public async Task<MyEntity> GetMyEntityAsync(int id)
{
  // Define a unique key for this method and its parameters.
  var key = $"MyEntity:{id}";
  var cache = Connection.GetDatabase();

  // Try to get the entity from the cache.
  var json = await cache.StringGetAsync(key).ConfigureAwait(false);
  var value = string.IsNullOrWhiteSpace(json)
                ? default(MyEntity)
                : JsonConvert.DeserializeObject<MyEntity>(json);

  if (value == null) // Cache miss
  {
    // If there's a cache miss, get the entity from the original store and cache it.
    // Code has been omitted because it is data store dependent.
    value = ...;

    // Avoid caching a null value.
    if (value != null)
    {
      // Put the item in the cache with a custom expiration time that
      // depends on how critical it is to have stale data.
      await cache.StringSetAsync(key, JsonConvert.SerializeObject(value)).ConfigureAwait(false);
      await cache.KeyExpireAsync(key, TimeSpan.FromMinutes(DefaultExpirationTimeInMinutes)).ConfigureAwait(false);
    }
  }

  return value;
}
```

> <span data-ttu-id="cf9ac-176">此示例使用 Redis 缓存访问存储并从缓存中检索信息。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-176">The examples use Redis Cache to access the store and retrieve information from the cache.</span></span> <span data-ttu-id="cf9ac-177">有关详细信息，请参阅[使用 Microsoft Azure Redis 缓存](/azure/redis-cache/cache-dotnet-how-to-use-azure-redis-cache)和[如何使用 Redis 缓存创建 Web 应用](/azure/redis-cache/cache-web-app-howto)</span><span class="sxs-lookup"><span data-stu-id="cf9ac-177">For more information, see [Using Microsoft Azure Redis Cache](/azure/redis-cache/cache-dotnet-how-to-use-azure-redis-cache) and [How to create a Web App with Redis Cache](/azure/redis-cache/cache-web-app-howto)</span></span>

<span data-ttu-id="cf9ac-178">如下所示，`UpdateEntityAsync` 方法演示如何在应用程序更改值时使缓存中的对象无效。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-178">The `UpdateEntityAsync` method shown below demonstrates how to invalidate an object in the cache when the value is changed by the application.</span></span> <span data-ttu-id="cf9ac-179">代码更新原始数据存储，然后从缓存中删除缓存的项。</span><span class="sxs-lookup"><span data-stu-id="cf9ac-179">The code updates the original data store and then removes the cached item from the cache.</span></span>

```csharp
public async Task UpdateEntityAsync(MyEntity entity)
{
    // Update the object in the original data store.
    await this.store.UpdateEntityAsync(entity).ConfigureAwait(false);

    // Invalidate the current cache object.
    var cache = Connection.GetDatabase();
    var id = entity.Id;
    var key = $"MyEntity:{id}"; // The key for the cached object.
    await cache.KeyDeleteAsync(key).ConfigureAwait(false); // Delete this key from the cache.
}
```

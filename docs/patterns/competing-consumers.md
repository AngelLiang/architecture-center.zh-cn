## <a name="context-and-problem"></a><span data-ttu-id="2b848-107">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="2b848-107">Context and problem</span></span>

<span data-ttu-id="2b848-111">在一段时间内，由于多种原因请求的数量会大幅度变化。</span><span class="sxs-lookup"><span data-stu-id="2b848-111">The number of requests can vary significantly over time for many reasons.</span></span> <span data-ttu-id="2b848-112">用户活动或来自多个租户的总请求数的突增可能会导致不可预测的工作负荷。</span><span class="sxs-lookup"><span data-stu-id="2b848-112">A sudden increase in user activity or aggregated data-ttu-id="2b848-114">此外，为处理请求而执行的工作的性质可能会有很大变化。</span><span class="sxs-lookup"><span data-stu-id="2b848-114">Additionally, the nature of the work performed to handle these requests might be highly variable.</span></span> <span data-ttu-id="2b848-115">运行使用者服务的单个实例可能导致该实例充满请求，或者消息系统可能由于来自应用程序的消息涌入而过载。</span><span class="sxs-lookup"><span data-stu-id="2b848-115">Using a single instance of the ## <a name="solution"></a><span data-ttu-id="2b848-119">解决方案</span><span class="sxs-lookup"><span data-stu-id="2b848-119">Solution</span></span>

<span data-ttu-id="2b848-120">使用消息队列来实现应用程序和使用者服务实例之间的信道。</span><span class="sxs-lookup"><span data-stu-id="2b848-120">Use a message queue to implement the communication channel between the application and the instances of the consumer service.</span></span> <span data-ttu-id="2b848-121">应用程序以消息的形式将请求发送到队列，使用者服务实例从队列接收消息并进行处理。</span><span class="sxs-lookup"><span data-stu-id="2b848-121">The application posts requests in the form of messages to the queue, and the consumer service instances receive messages from the queue and process them.</span></span> <span data-ttu-id="2b848-122">此方法可让使用者服务实例的相同池处理来自应用程序实例的消息。</span><span class="sxs-lookup"><span data-stu-id="2b848-122">This approach enables the same pool of consumer service instances to handle messages from any instance of the application.</span></span> <span data-ttu-id="2b848-123">该图说明了如何使用消息队列将工作分布到服务实例。</span><span class="sxs-lookup"><span data-stu-id="2b848-123">The figure illustrates using a message queue to distribute work to instances of a service.</span></span>

![使用消息队列将工作分布到服务实例](./_images/competing-consumers-diagram.png)

<span data-ttu-id="2b848-125">此解决方案具有以下优点：</span><span class="sxs-lookup"><span data-stu-id="2b848-125">This solution has the following benefits:</span></span>

- <span data-ttu-id="2b848-126">它提供了一个负载调节系统，可以处理应用程序实例发送的大幅度变化的请求量。</span><span class="sxs-lookup"><span data-stu-id="2b848-126">It provides a load-leveled system that can handle wide variations in the volume of requests sent by application instances.</span></span> <span data-ttu-id="2b848-127">队列充当应用程序实例和使用者服务实例之间的缓冲区。</span><span class="sxs-lookup"><span data-stu-id="2b848-127">The queue acts as a buffer between the application instances and the consumer service instances.</span></span> <span data-ttu-id="2b848-128">这有助于尽量减少对应用程序和服务实例的可用性和响应性的影响，如[基于队列的负载调节模式](./queue-based-load-leveling.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="2b848-128">This can help to minimize the impact on availability and responsiveness for both the application and the service instances, as described by the [Queue-based Load Leveling pattern](./queue-based-load-leveling.md).</span></span> <span data-ttu-id="2b848-129">处理需要长时间运行处理的消息时不会阻止使用者服务的其他实例同时处理其他消息。</span><span class="sxs-lookup"><span data-stu-id="2b848-129">Handling a message that requires some long-running processing doesn't prevent other messages from being handled concurrently by other instances of the consumer service.</span></span>

- <span data-ttu-id="2b848-130">它提高了可靠性。</span><span class="sxs-lookup"><span data-stu-id="2b848-130">It improves reliability.</span></span> <span data-ttu-id="2b848-131">如果生成者直接与使用者通信，而不使用这种模式且不对使用者进行监视，则消息很可能丢失或未能处理（如果使用者失败）。</span><span class="sxs-lookup"><span data-stu-id="2b848-131">If a producer communicates directly with a consumer instead of using this pattern, but doesn't monitor the consumer, there's a high probability that messages could be lost or fail to be processed if the consumer fails.</span></span> <span data-ttu-id="2b848-132">在此模式中，消息不会发送到特定服务实例。</span><span class="sxs-lookup"><span data-stu-id="2b848-132">In this pattern, messages aren't sent to a specific service instance.</span></span> <span data-ttu-id="2b848-133">失败的服务实例不会阻止生成者，并且任何工作服务实例都可处理消息。</span><span class="sxs-lookup"><span data-stu-id="2b848-133">A failed service instance won't block a producer, and messages can be processed by any working service instance.</span></span>

- <span data-ttu-id="2b848-134">它不需要使用者之间或生成者与使用者实例之间的复杂协调。</span><span class="sxs-lookup"><span data-stu-id="2b848-134">It doesn't require complex coordination between the consumers, or between the producer and the consumer instances.</span></span> <span data-ttu-id="2b848-135">消息队列可确保每条消息至少传送一次。</span><span class="sxs-lookup"><span data-stu-id="2b848-135">The message queue ensures that each message is delivered at least once.</span></span>

- <span data-ttu-id="2b848-136">可缩放。</span><span class="sxs-lookup"><span data-stu-id="2b848-136">It's scalable.</span></span> <span data-ttu-id="2b848-137">消息数量波动时，系统可以动态地增加或减少使用者服务实例的数量。</span><span class="sxs-lookup"><span data-stu-id="2b848-137">The system can dynamically increase or decrease the number of instances of the consumer service as the volume of messages fluctuates.</span></span>

- <span data-ttu-id="2b848-138">如果消息队列提供事务读取操作，则可以提高复原能力。</span><span class="sxs-lookup"><span data-stu-id="2b848-138">It can improve resiliency if the message queue provides transactional read operations.</span></span> <span data-ttu-id="2b848-139">如果使用者服务实例读取和处理消息（作为事务操作的一部分），并且使用者服务实例失败，则该模式可以确保消息将返回到队列由另一使用者服务实例进行选取并处理。</span><span class="sxs-lookup"><span data-stu-id="2b848-139">If a consumer service instance reads and processes the message as part of a transactional operation, and the consumer service instance fails, this pattern can ensure that the message will be returned to the queue to be picked up and handled by another instance of the consumer service.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="2b848-140">问题和注意事项</span><span class="sxs-lookup"><span data-stu-id="2b848-140">Issues and considerations</span></span>

# <a name="throttling-pattern"></a><span data-ttu-id="719a2-104">限制模式</span><span class="sxs-lookup"><span data-stu-id="719a2-104">Throttling pattern</span></span>

[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="719a2-105">控制应用程序实例、单个租户或整个服务对资源的消耗。</span><span class="sxs-lookup"><span data-stu-id="719a2-105">Control the consumption of resources used by an instance of an application, an individual tenant, or an entire service.</span></span> <span data-ttu-id="719a2-106">这样，即使需求增加给资源带来了极大的负载，系统也可以继续运行以满足服务级别协议。</span><span class="sxs-lookup"><span data-stu-id="719a2-106">This can allow the system to continue to function and meet service level agreements, even when an increase in demand places an extreme load on resources.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="719a2-107">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="719a2-107">Context and problem</span></span>

<span data-ttu-id="719a2-108">云应用程序上的负载通常根据活动用户的数目或其执行的活动的类型随时间而变。</span><span class="sxs-lookup"><span data-stu-id="719a2-108">The load on a cloud application typically varies over time based on the number of active users or the types of activities they are performing.</span></span> <span data-ttu-id="719a2-109">例如，在营业时间内可能有更多用户处于活动状态，而系统在每个月末可能需要执行计算成本高昂的分析。</span><span class="sxs-lookup"><span data-stu-id="719a2-109">For example, more users are likely to be active during business hours, or the system might be required to perform computationally expensive analytics at the end of each month.</span></span> <span data-ttu-id="719a2-110">还可能会突然出现意外的活动量激增。</span><span class="sxs-lookup"><span data-stu-id="719a2-110">There might also be sudden and unanticipated bursts in activity.</span></span> <span data-ttu-id="719a2-111">如果系统的处理要求超出了可用资源的容量，则系统性能将降低甚至会失败。</span><span class="sxs-lookup"><span data-stu-id="719a2-111">If the processing requirements of the system exceed the capacity of the resources that are available, it'll suffer from poor performance and can even fail.</span></span> <span data-ttu-id="719a2-112">如果系统必须满足约定的服务级别，则这样的故障可能不可接受。</span><span class="sxs-lookup"><span data-stu-id="719a2-112">If the system has to meet an agreed level of service, such failure could be unacceptable.</span></span>

<span data-ttu-id="719a2-113">有许多策略可用于处理云中的各种负载，具体取决于应用程序的业务目标。</span><span class="sxs-lookup"><span data-stu-id="719a2-113">There're many strategies available for handling varying load in the cloud, depending on the business goals for the application.</span></span> <span data-ttu-id="719a2-114">其中一个策略是使用自动缩放随时将预配的资源与用户需求进行匹配。</span><span class="sxs-lookup"><span data-stu-id="719a2-114">One strategy is to use autoscaling to match the provisioned resources to the user needs at any given time.</span></span> <span data-ttu-id="719a2-115">这可以持续满足用户需求，同时优化运行成本。</span><span class="sxs-lookup"><span data-stu-id="719a2-115">This has the potential to consistently meet user demand, while optimizing running costs.</span></span> <span data-ttu-id="719a2-116">不过，虽然自动缩放可以触发附加资源的预配，但此预配不是即时的。</span><span class="sxs-lookup"><span data-stu-id="719a2-116">However, while autoscaling can trigger the provisioning of additional resources, this provisioning isn't immediate.</span></span> <span data-ttu-id="719a2-117">如果需求增长快速，则可能有一段时间会出现资源不足。</span><span class="sxs-lookup"><span data-stu-id="719a2-117">If demand grows quickly, there can be a window of time where there's a resource deficit.</span></span>

## <a name="solution"></a><span data-ttu-id="719a2-118">解决方案</span><span class="sxs-lookup"><span data-stu-id="719a2-118">Solution</span></span>

<span data-ttu-id="719a2-119">自动缩放的一个替代策略是仅允许应用程序使用某个限制内的资源，当达到该限制时会对资源进行限制。</span><span class="sxs-lookup"><span data-stu-id="719a2-119">An alternative strategy to autoscaling is to allow applications to use resources only up to a limit, and then throttle them when this limit is reached.</span></span> <span data-ttu-id="719a2-120">系统应当监视其对资源的使用情况，以便在使用量超出阈值时可以限制来自一个或多个用户的请求。</span><span class="sxs-lookup"><span data-stu-id="719a2-120">The system should monitor how it's using resources so that, when usage exceeds the threshold, it can throttle requests from one or more users.</span></span> <span data-ttu-id="719a2-121">这将使系统可以继续正常工作并满足所实施的所有服务级别协议 (SLA)。</span><span class="sxs-lookup"><span data-stu-id="719a2-121">This will enable the system to continue functioning and meet any service level agreements (SLAs) that are in place.</span></span> <span data-ttu-id="719a2-122">有关监视资源使用情况的详细信息，请参阅[检测和遥测指南](https://msdn.microsoft.com/library/dn589775.aspx)。</span><span class="sxs-lookup"><span data-stu-id="719a2-122">For more information on monitoring resource usage, see the [Instrumentation and Telemetry Guidance](https://msdn.microsoft.com/library/dn589775.aspx).</span></span>

<span data-ttu-id="719a2-123">系统可以实现多个限制策略，包括：</span><span class="sxs-lookup"><span data-stu-id="719a2-123">The system could implement several throttling strategies, including:</span></span>

- <span data-ttu-id="719a2-124">如果用户在给定时间段内访问系统 API 的频率超过了每秒 n 次，则拒绝来自该用户的请求。</span><span class="sxs-lookup"><span data-stu-id="719a2-124">Rejecting requests from an individual user who's already accessed system APIs more than n times per second over a given period of time.</span></span> <span data-ttu-id="719a2-125">这要求系统对运行应用程序的每个租户或用户使用的资源量进行计量。</span><span class="sxs-lookup"><span data-stu-id="719a2-125">This requires the system to meter the use of resources for each tenant or user running an application.</span></span> <span data-ttu-id="719a2-126">有关详细信息，请参阅 [Service Metering Guidance](https://msdn.microsoft.com/library/dn589796.aspx)（服务计量指南）。</span><span class="sxs-lookup"><span data-stu-id="719a2-126">For more information, see the [Service Metering Guidance](https://msdn.microsoft.com/library/dn589796.aspx).</span></span>

- <span data-ttu-id="719a2-127">禁用所选非基本服务的功能或将其降级，以便基本服务可以使用充足的资源畅通运行。</span><span class="sxs-lookup"><span data-stu-id="719a2-127">Disabling or degrading the functionality of selected nonessential services so that essential services can run unimpeded with sufficient resources.</span></span> <span data-ttu-id="719a2-128">例如，如果应用程序对视频输出进行流式处理，则可以切换到较低的分辨率。</span><span class="sxs-lookup"><span data-stu-id="719a2-128">For example, if the application is streaming video output, it could switch to a lower resolution.</span></span>

- <span data-ttu-id="719a2-129">使用负载调节来均衡活动量（[基于队列的负载调节模式](./queue-based-load-leveling.md)中更详细地介绍了此方法）。</span><span class="sxs-lookup"><span data-stu-id="719a2-129">Using load leveling to smooth the volume of activity (this approach is covered in more detail by the [Queue-based Load Leveling pattern](./queue-based-load-leveling.md)).</span></span> <span data-ttu-id="719a2-130">在多租户环境中，此方法会降低每个租户的性能。</span><span class="sxs-lookup"><span data-stu-id="719a2-130">In a multi-tenant environment, this approach will reduce the performance for every tenant.</span></span> <span data-ttu-id="719a2-131">如果系统必须支持混合使用具有不同 SLA 的租户，则高值租户的工作可以立即执行。</span><span class="sxs-lookup"><span data-stu-id="719a2-131">If the system must support a mix of tenants with different SLAs, the work for high-value tenants might be performed immediately.</span></span> <span data-ttu-id="719a2-132">其他租户的请求可以推迟，并在积压工作得到缓解时再进行处理。</span><span class="sxs-lookup"><span data-stu-id="719a2-132">Requests for other tenants can be held back, and handled when the backlog has eased.</span></span> <span data-ttu-id="719a2-133">可以使用[优先级队列模式][]来帮助实现此方法。</span><span class="sxs-lookup"><span data-stu-id="719a2-133">The [Priority Queue pattern][] could be used to help implement this approach.</span></span>

- <span data-ttu-id="719a2-134">推迟以低优先级应用程序或租户的身份执行的操作。</span><span class="sxs-lookup"><span data-stu-id="719a2-134">Deferring operations being performed on behalf of lower priority applications or tenants.</span></span> <span data-ttu-id="719a2-135">可以暂停或限制这些操作，但是要生成一个异常来通知租户系统正忙并且应当稍后重试该操作。</span><span class="sxs-lookup"><span data-stu-id="719a2-135">These operations can be suspended or limited, with an exception generated to inform the tenant that the system is busy and that the operation should be retried later.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="719a2-156">问题和注意事项</span><span class="sxs-lookup"><span data-stu-id="719a2-156">Issues and considerations</span></span>

<span data-ttu-id="719a2-157">在决定如何实现此模式时，应考虑以下几点：</span><span class="sxs-lookup"><span data-stu-id="719a2-157">You should consider the following points when deciding how to implement this pattern:</span></span>

- <span data-ttu-id="719a2-160">限制必须快速执行。</span><span class="sxs-lookup"><span data-stu-id="719a2-160">Throttling must be performed quickly.</span></span> <span data-ttu-id="719a2-161">系统必须能够检测活动量增加并相应地作出反应。</span><span class="sxs-lookup"><span data-stu-id="719a2-161">The system must be capable of detecting an increase in activity and react accordingly.</span></span> <span data-ttu-id="719a2-162">在负载减轻后，系统还必须能够快速恢复其原始状态。</span><span class="sxs-lookup"><span data-stu-id="719a2-162">The system must also be able to revert to its original state quickly after the load has eased.</span></span> <span data-ttu-id="719a2-163">这需要持续捕获并监视相应的性能数据。</span><span class="sxs-lookup"><span data-stu-id="719a2-163">This requires that the appropriate performance data is continually captured and monitored.</span></span>

- <span data-ttu-id="719a2-164">如果某个服务需要暂时拒绝用户请求，则它应当返回一个特定的错误代码，以便客户端应用程序能够了解拒绝执行操作的原因是由于受到限制。</span><span class="sxs-lookup"><span data-stu-id="719a2-164">If a service needs to temporarily deny a user request, it should return a specific error code so the client application understands that the reason for the refusal to perform an operation is due to throttling.</span></span> <span data-ttu-id="719a2-165">客户端应用程序可以等待一段时间，然后重试请求。</span><span class="sxs-lookup"><span data-stu-id="719a2-165">The client application can wait for a period before retrying the request.</span></span>

- <span data-ttu-id="719a2-168">如果在系统进行自动缩放期间使用限制作为临时措施，并且资源需求增长非常快速，则&mdash;即使在受限制模式下运行，系统也可能无法继续正常工作。</span><span class="sxs-lookup"><span data-stu-id="719a2-168">If throttling is being used as a temporary measure while a system autoscales, and if resource demands grow very quickly, the system might not be able to continue functioning&mdash;even when operating in a throttled mode.</span></span> <span data-ttu-id="719a2-169">如果这不可接受，请考虑维护更大的容量预留并配置更积极的自动缩放。</span><span class="sxs-lookup"><span data-stu-id="719a2-169">If this isn't acceptable, consider maintaining larger capacity reserves and configuring more aggressive autoscaling.</span></span>

# <a name="publisher-subscriber-pattern"></a><span data-ttu-id="69658-104">发布者-订阅者模式</span><span class="sxs-lookup"><span data-stu-id="69658-104">Publisher-Subscriber pattern</span></span>

<span data-ttu-id="69658-105">使应用程序能够以异步方式向多个感兴趣的使用者公布事件，而无需将发送方与接收方耦合。</span><span class="sxs-lookup"><span data-stu-id="69658-105">Enable an application to announce events to multiple interested consumers aynchronously, without coupling the senders to the receivers.</span></span>

<span data-ttu-id="69658-106">**也称为**：发布/订阅消息传送</span><span class="sxs-lookup"><span data-stu-id="69658-106">**Also called**: Pub/sub messaging</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="69658-107">上下文和问题</span><span class="sxs-lookup"><span data-stu-id="69658-107">Context and problem</span></span>

<span data-ttu-id="69658-108">在基于云的分布式应用程序中，在发生事件时，系统组件通常需要向其他组件提供信息。</span><span class="sxs-lookup"><span data-stu-id="69658-108">In cloud-based and distributed applications, components of the system often need to provide information to other components as events happen.</span></span>

<span data-ttu-id="69658-109">异步消息传送是将发件者与使用者解耦的有效方式，可避免阻止发送者等待响应。</span><span class="sxs-lookup"><span data-stu-id="69658-109">Asynchronous messaging is an effective way to decouple senders from consumers, and avoid blocking the sender to wait for a response.</span></span> <span data-ttu-id="69658-110">但是，对每个使用者使用专用消息队列不能让多个使用者有效进行缩放。</span><span class="sxs-lookup"><span data-stu-id="69658-110">However, using a dedicated message queue for each consumer does not effectively scale to many consumers.</span></span> <span data-ttu-id="69658-111">此外，某些使用者可能仅对一部分信息感兴趣。</span><span class="sxs-lookup"><span data-stu-id="69658-111">Also, some of the consumers might be interested in only a subset of the information.</span></span> <span data-ttu-id="69658-112">发送者如何在不知道使用者标识的情况下，向所有感兴趣的使用者公布事件？</span><span class="sxs-lookup"><span data-stu-id="69658-112">How can the sender announce events to all interested consumers without knowing their identities?</span></span>

## <a name="solution"></a><span data-ttu-id="69658-113">解决方案</span><span class="sxs-lookup"><span data-stu-id="69658-113">Solution</span></span>

<span data-ttu-id="69658-114">引入一个包括以下功能的异步消息传送子系统：</span><span class="sxs-lookup"><span data-stu-id="69658-114">Introduce an asynchronous messaging subsystem that includes the following:</span></span>

- <span data-ttu-id="69658-115">发送者使用的输入消息传送通道。</span><span class="sxs-lookup"><span data-stu-id="69658-115">An input messaging channel used by the sender.</span></span> <span data-ttu-id="69658-116">发送者使用已知的消息格式将事件打包成消息，然后通过输入通道发送这些消息。</span><span class="sxs-lookup"><span data-stu-id="69658-116">The sender packages events into messages, using a known message format, and sends these messages via the input channel.</span></span> <span data-ttu-id="69658-117">在此模式中，发送者也称为“发布者”。</span><span class="sxs-lookup"><span data-stu-id="69658-117">The sender in this pattern is also called the *publisher*.</span></span>

  > [!NOTE]
  > <span data-ttu-id="69658-118">消息是数据包。</span><span class="sxs-lookup"><span data-stu-id="69658-118">A *message* is a packet of data.</span></span> <span data-ttu-id="69658-119">事件是一条消息，告知其他组件已发生某项更改或操作。</span><span class="sxs-lookup"><span data-stu-id="69658-119">An *event* is a message that notifies other components about a change or an action that has taken place.</span></span>

- <span data-ttu-id="69658-120">每个使用者有一个输出消息传送通道。</span><span class="sxs-lookup"><span data-stu-id="69658-120">One output messaging channel per consumer.</span></span> <span data-ttu-id="69658-121">使用者称为“订阅者”。</span><span class="sxs-lookup"><span data-stu-id="69658-121">The consumers are known as *subscribers*.</span></span>

- <span data-ttu-id="69658-122">一个机制，用于将输入通道中的每条消息复制到对该消息感兴趣的所有订阅者的输出通道。</span><span class="sxs-lookup"><span data-stu-id="69658-122">A mechanism for copying each message from the input channel to the output channels for all subscribers interested in that message.</span></span> <span data-ttu-id="69658-123">此操作通常由消息中转站或事件总线等中介处理。</span><span class="sxs-lookup"><span data-stu-id="69658-123">This operation is typically handled by a intermediary such as a message broker or event bus.</span></span>

<span data-ttu-id="69658-126">发布/订阅消息传送具有以下优势：</span><span class="sxs-lookup"><span data-stu-id="69658-126">Pub/sub messaging has the following benefits:</span></span>

- <span data-ttu-id="69658-127">解耦仍然需要通信的子系统。</span><span class="sxs-lookup"><span data-stu-id="69658-127">It decouples subsystems that still need to communicate.</span></span> <span data-ttu-id="69658-128">可以独立管理子系统，即使一个或多个接收者处于脱机状态，也能正确管理消息。</span><span class="sxs-lookup"><span data-stu-id="69658-128">Subsystems can be managed independently, and messages can be properly managed even if one or more receivers are offline.</span></span>

- <span data-ttu-id="69658-129">提高可伸缩性，改善发送者的响应能力。</span><span class="sxs-lookup"><span data-stu-id="69658-129">It increases scalability and improves responsiveness of the sender.</span></span> <span data-ttu-id="69658-130">发送者可以快速将一条消息发送到输入通道，然后恢复其核心处理责任。</span><span class="sxs-lookup"><span data-stu-id="69658-130">The sender can quickly send a single message to the input channel, then return to its core processing responsibilities.</span></span> <span data-ttu-id="69658-131">消息传送基础结构负责确保将消息传送到感兴趣的订阅者。</span><span class="sxs-lookup"><span data-stu-id="69658-131">The messaging infrastructure is responsible for ensuring messages are delivered to interested subscribers.</span></span>

- <span data-ttu-id="69658-132">它提高了可靠性。</span><span class="sxs-lookup"><span data-stu-id="69658-132">It improves reliability.</span></span> <span data-ttu-id="69658-133">异步消息传送可以帮助应用程序在负载增大的情况下继续保持平稳运行，并更有效地处理间歇性故障。</span><span class="sxs-lookup"><span data-stu-id="69658-133">Asynchronous messaging helps applications continue to run smoothly under increased loads and handle intermittent failures more effectively.</span></span>

- <span data-ttu-id="69658-134">它允许延迟或计划的处理。</span><span class="sxs-lookup"><span data-stu-id="69658-134">It allows for deferred or scheduled processing.</span></span> <span data-ttu-id="69658-135">订阅者可以等到在非高峰期拾取消息，或者可以根据特定的计划路由或处理消息。</span><span class="sxs-lookup"><span data-stu-id="69658-135">Subscribers can wait to pick up messages until off-peak hours, or messages can be routed or processed according to a specific schedule.</span></span>

- <span data-ttu-id="69658-136">这样可以简化使用不同平台、编程语言或通信协议的系统之间的集成，以及本地系统与云中运行的应用程序之间的集成。</span><span class="sxs-lookup"><span data-stu-id="69658-136">It enables simpler integration between systems using different platforms, programming languages, or communication protocols, as well as between on-premises systems and applications running in the cloud.</span></span>

- <span data-ttu-id="69658-137">它简化了整个企业中的异步工作流。</span><span class="sxs-lookup"><span data-stu-id="69658-137">It facilitates asynchronous workflows across an enterprise.</span></span>

- <span data-ttu-id="69658-138">它改善了可测试性。</span><span class="sxs-lookup"><span data-stu-id="69658-138">It improves testability.</span></span> <span data-ttu-id="69658-139">在执行整个集成测试策略过程中，可以监视通道，并可以检查或记录消息。</span><span class="sxs-lookup"><span data-stu-id="69658-139">Channels can be monitored and messages can be inspected or logged as part of an overall integration test strategy.</span></span>

- <span data-ttu-id="69658-140">它为应用程序提供关注点分离。</span><span class="sxs-lookup"><span data-stu-id="69658-140">It provides separation of concerns for your applications.</span></span> <span data-ttu-id="69658-141">每个应用程序可以注重自身的核心功能，而消息传送基础结构可以处理所需的一切工作来可靠地将消息路由到多个使用者。</span><span class="sxs-lookup"><span data-stu-id="69658-141">Each application can focus on its core capabilities, while the messaging infrastructure handles everything required to reliably route messages to multiple consumers.</span></span>

## <a name="when-to-use-this-pattern"></a><span data-ttu-id="69658-189">何时使用此模式</span><span class="sxs-lookup"><span data-stu-id="69658-189">When to use this pattern</span></span>

- <span data-ttu-id="69658-192">应用程序需要与一个或多个独立开发的、可能使用不同平台、编程语言和通信协议的应用程序或服务通信。</span><span class="sxs-lookup"><span data-stu-id="69658-192">An application needs to communicate with one or more independently-developed applications or services, which may use different platforms, programming languages, and communication protocols.</span></span>

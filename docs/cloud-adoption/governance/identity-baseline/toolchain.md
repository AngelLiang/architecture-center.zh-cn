# <a name="identity-baseline-tools-in-azure"></a><span data-ttu-id="47b01-103">Azure 中的标识基线工具</span><span class="sxs-lookup"><span data-stu-id="47b01-103">Identity Baseline tools in Azure</span></span>

<span data-ttu-id="47b01-104">[基线](overview.md)是[云治理的五项规则](../governance-disciplines.md)之一。</span><span class="sxs-lookup"><span data-stu-id="47b01-104">[Identity Baseline](overview.md) is one of the [Five Disciplines of Cloud Governance](../governance-disciplines.md).</span></span> <span data-ttu-id="47b01-105">此规则侧重于建立策略的方法，以确保用户标识的一致性和连续性，而不考虑托管应用程序或工作负荷的云提供程序。</span><span class="sxs-lookup"><span data-stu-id="47b01-105">This discipline focuses on ways of establishing policies that ensure consistency and continuity of user identities regardless of the cloud provider that hosts the application or workload.</span></span>

<span data-ttu-id="47b01-106">有关混合标识的发现指南中包括以下工具。</span><span class="sxs-lookup"><span data-stu-id="47b01-106">The following tools are included in the discovery guide on Hybrid Identity.</span></span>

<span data-ttu-id="47b01-107">**Active Directory（本地）：** Active Directory 是在企业中最常用于存储和验证用户凭据的标识提供者。</span><span class="sxs-lookup"><span data-stu-id="47b01-107">**Active Directory (on-premises):** Active Directory is the identity provider most frequently used in the enterprise to store and validate user credentials.</span></span>

<span data-ttu-id="47b01-108">**Azure Active Directory：** 等效于 Active Directory 的软件即服务 (SaaS)，能够与本地 Active Directory 联合。</span><span class="sxs-lookup"><span data-stu-id="47b01-108">**Azure Active Directory:** A software as a service (SaaS) equivalent to Active Directory, capable of federating with an on-premises Active Directory.</span></span>

<span data-ttu-id="47b01-109">**Active Directory (IaaS)：** 在 Azure 中的虚拟机内运行的 Active Directory 应用程序实例。</span><span class="sxs-lookup"><span data-stu-id="47b01-109">**Active Directory (IaaS):** An instance of the Active Directory application running in a virtual machine in Azure.</span></span>

<span data-ttu-id="47b01-110">标识是用于 IT 安全性的控制平面。</span><span class="sxs-lookup"><span data-stu-id="47b01-110">Identity is the control plane for IT security.</span></span> <span data-ttu-id="47b01-111">因此，身份验证是组织对云的访问防护。</span><span class="sxs-lookup"><span data-stu-id="47b01-111">So authentication is an organization’s access guard to the cloud.</span></span> <span data-ttu-id="47b01-112">组织需要一个标识控制平面，以便增强其安全性并使其云应用免受入侵者攻击。</span><span class="sxs-lookup"><span data-stu-id="47b01-112">Organizations need an identity control plane that strengthens their security and keeps their cloud apps safe from intruders.</span></span>

## <a name="cloud-authentication"></a><span data-ttu-id="47b01-113">云身份验证</span><span class="sxs-lookup"><span data-stu-id="47b01-113">Cloud authentication</span></span>

<span data-ttu-id="47b01-114">对于希望将应用移动到云的组织来说，应首先考虑选择正确的身份验证方式。</span><span class="sxs-lookup"><span data-stu-id="47b01-114">Choosing the correct authentication method is the first concern for organizations wanting to move their apps to the cloud.</span></span>

<span data-ttu-id="47b01-115">选择此方法时，Azure AD 会处理用户的登录过程。</span><span class="sxs-lookup"><span data-stu-id="47b01-115">When you choose this method, Azure AD handles users' sign-in process.</span></span> <span data-ttu-id="47b01-116">结合使用无缝单一登录 (SSO)，用户可以登录到云应用，无需重新输入其凭据。</span><span class="sxs-lookup"><span data-stu-id="47b01-116">Coupled with seamless single sign-on (SSO), users can sign in to cloud apps without having to reenter their credentials.</span></span> <span data-ttu-id="47b01-117">如果使用的是云身份验证，可以从以下两个选项中选择：</span><span class="sxs-lookup"><span data-stu-id="47b01-117">With cloud authentication, you can choose from two options:</span></span>

<span data-ttu-id="47b01-118">**Azure AD 密码哈希同步**：这是在 Azure AD 中为本地目录对象启用身份验证的最简单方法。</span><span class="sxs-lookup"><span data-stu-id="47b01-118">**Azure AD password hash synchronization**: The simplest way to enable authentication for on-premises directory objects in Azure AD.</span></span> <span data-ttu-id="47b01-119">此方法还可以与任何在本地服务器出现故障的情况下用作备份故障转移身份验证方法的方法结合使用。</span><span class="sxs-lookup"><span data-stu-id="47b01-119">This method can also be used with any method as a back-up failover authentication method in case your on-premises server goes down.</span></span>

<span data-ttu-id="47b01-120">**Azure AD 直通身份验证**：通过使用在一个或多个本地服务器上运行的软件代理，为 Azure AD 身份验证服务提供持久密码验证。</span><span class="sxs-lookup"><span data-stu-id="47b01-120">**Azure AD Pass-through Authentication**: Provides a persistent password validation for Azure AD authentication services by using a software agent that runs on one or more on-premises servers.</span></span>

> [!NOTE]
> <span data-ttu-id="47b01-121">具有即时强制本地用户帐户状态、密码策略和登录小时数生效的安全要求的公司应考虑直通身份验证方法。</span><span class="sxs-lookup"><span data-stu-id="47b01-121">Companies with a security requirement to immediately enforce on-premises user account states, password policies, and sign-in hours should consider the pass-through Authentication method.</span></span>

<span data-ttu-id="47b01-122">**联合身份验证：**</span><span class="sxs-lookup"><span data-stu-id="47b01-122">**Federated authentication:**</span></span>

<span data-ttu-id="47b01-123">选择此方法时，Azure AD 将身份验证过程传给单独的受信任身份验证系统（例如本地 Active Directory 联合身份验证服务 (AD FS) 或受信任的第三方联合身份验证提供程序）来验证用户的密码。</span><span class="sxs-lookup"><span data-stu-id="47b01-123">When you choose this method, Azure AD passes the authentication process to a separate trusted authentication system, such as on-premises Active Directory Federation Services (AD FS) or a trusted third-party federation provider, to validate the user’s password.</span></span>

<span data-ttu-id="47b01-124">[为 Azure Active Directory 选择正确的身份验证方法](/azure/security/azure-ad-choose-authn)一文包含决策树，以帮助为组织选择最佳解决方案。</span><span class="sxs-lookup"><span data-stu-id="47b01-124">The article [choosing the right authentication method for Azure Active Directory](/azure/security/azure-ad-choose-authn) contains a decision tree to help you choose the best solution for your organization.</span></span>

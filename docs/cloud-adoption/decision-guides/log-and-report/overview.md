### <a name="cloud-native"></a><span data-ttu-id="5b776-139">云原生</span><span class="sxs-lookup"><span data-stu-id="5b776-139">Cloud native</span></span>

<span data-ttu-id="5b776-140">如果组织当前缺少已建立的日志记录和报告系统，或者如果计划的云部署不需要与现有本地或其他外部监视系统集成，则云原生 SaaS 解决方案是最简单的选择。</span><span class="sxs-lookup"><span data-stu-id="5b776-140">If your organization currently lacks established logging and reporting systems, or if your planned cloud deployment does not need to be integrated with existing on-premises or other external monitoring systems, a cloud native SaaS solution is the simplest choice.</span></span>

<span data-ttu-id="5b776-141">在此方案中，日志数据会记录并存储在与工作负荷相同的云环境中，同时作为云平台的一部分来提供用于处理信息并呈现给 IT 人员的日志记录和报告工具。</span><span class="sxs-lookup"><span data-stu-id="5b776-141">In this scenario, log data is recorded and stored in the same cloud environment as your workload, while the logging and reporting tools that process and surface information to IT staff are offered as part of the cloud platform.</span></span>

<span data-ttu-id="5b776-142">云原生日志记录解决方案可以专门针对每个订阅或工作负荷来实现，以实现较小部署或实验部署，并采用集中方式进行组织以监视整个云资产中的日志数据。</span><span class="sxs-lookup"><span data-stu-id="5b776-142">Cloud native logging solutions can be implemented ad hoc per subscription or workload for smaller or experimental deployments and are organized in a centralized manner to monitor log data across your entire cloud estate.</span></span>

<span data-ttu-id="5b776-143">**云原生假设**。</span><span class="sxs-lookup"><span data-stu-id="5b776-143">**Cloud native assumptions**.</span></span> <span data-ttu-id="5b776-144">使用云原生日志记录和报告系统假设以下情况：</span><span class="sxs-lookup"><span data-stu-id="5b776-144">Using a cloud native logging and reporting system assumes the following:</span></span>

### <a name="on-premises-extension"></a><span data-ttu-id="5b776-147">本地扩展</span><span class="sxs-lookup"><span data-stu-id="5b776-147">On-premises extension</span></span>

<span data-ttu-id="5b776-148">在需要将云遥测数据与不支持混合日志记录和报告，或是支持通过最少量的重新开发迁移应用程序和服务的本地系统集成的方案中，需要将监视代理部署到 VM，这些代理会将日志数据直接发送到本地系统，而不是将其存储在云环境中。</span><span class="sxs-lookup"><span data-stu-id="5b776-148">In scenarios where you need to integrate cloud telemetry with on-premises systems that do not support hybrid logging and reporting, or support the migration of applications and services with a minimum amount of redevelopment, you will need to deploy monitoring agents to VMs that will send log data directly to your on-premises systems, rather than storing it in the cloud environment.</span></span>

<span data-ttu-id="5b776-149">若要支持这种方法，云资源需要能够通过[混合网络](../software-defined-network/hybrid.md)和[云托管的域服务](../identity/overview.md#cloud-hosted-domain-services)的组合，直接与本地系统进行通信。</span><span class="sxs-lookup"><span data-stu-id="5b776-149">In order to support this approach, your cloud resources will need to be able to communicate directly with your on-premises systems through a combination of [hybrid networking](../software-defined-network/hybrid.md) and [cloud hosted domain services](../identity/overview.md#cloud-hosted-domain-services).</span></span> <span data-ttu-id="5b776-150">实施此方法时，云虚拟网络充当本地环境的网络扩展。</span><span class="sxs-lookup"><span data-stu-id="5b776-150">With this in place, the cloud virtual network functions as a network extension of the on-premises environment.</span></span> <span data-ttu-id="5b776-151">因此，云托管的工作负荷可以直接与本地日志记录和报告系统进行通信。</span><span class="sxs-lookup"><span data-stu-id="5b776-151">Therefore, cloud hosted workloads can communicate directly with your on-premises logging and reporting system.</span></span>

<span data-ttu-id="5b776-152">此方法利用对监视工具的现有投资，只需对任何云部署的应用程序或服务进行有限修改。</span><span class="sxs-lookup"><span data-stu-id="5b776-152">This approach capitalizes on your existing investment in monitoring tooling with limited modification to any cloud-deployed applications or services.</span></span> <span data-ttu-id="5b776-153">这通常是在直接迁移过程支持监视的最快方法。</span><span class="sxs-lookup"><span data-stu-id="5b776-153">This is often the fastest approach to support monitoring during a lift-and-shift migration.</span></span> <span data-ttu-id="5b776-154">但是，它不会捕获由基于云的 PaaS 和 SaaS 资源生成的日志数据，并且会忽略由云平台自身生成的任何与 VM 相关的日志（如 VM 状态）。</span><span class="sxs-lookup"><span data-stu-id="5b776-154">However, it won’t capture log data produced by cloud-based PaaS and SaaS resources, and it will omit any VM-related logs generated by the cloud platform itself such as VM status.</span></span> <span data-ttu-id="5b776-155">因此，此模式应是临时解决方案，直到实现更全面的混合监视解决方案。</span><span class="sxs-lookup"><span data-stu-id="5b776-155">As a result, this pattern should be a temporary solution until a more comprehensive hybrid monitoring solution is implemented.</span></span>

<span data-ttu-id="5b776-156">仅本地假设：</span><span class="sxs-lookup"><span data-stu-id="5b776-156">On-premises only assumptions:</span></span>

- <span data-ttu-id="5b776-159">基于云的应用程序可以将遥测数据直接提交到本地日志记录系统，或是可以向工作负荷 VM 部署向本地提交的监视代理。</span><span class="sxs-lookup"><span data-stu-id="5b776-159">Your cloud-based applications can submit telemetry directly to your on-premises logging systems or monitoring agents that submit to on-premises can be deployed to workload VMs.</span></span>

### <a name="gateway-aggregation"></a><span data-ttu-id="5b776-161">网关聚合</span><span class="sxs-lookup"><span data-stu-id="5b776-161">Gateway aggregation</span></span>

<span data-ttu-id="5b776-163">网关服务会部署到云提供程序。</span><span class="sxs-lookup"><span data-stu-id="5b776-163">A gateway service is deployed to your cloud provider.</span></span> <span data-ttu-id="5b776-164">随后配置相关应用程序和服务以将遥测数据提交到网关而不是默认日志记录系统。</span><span class="sxs-lookup"><span data-stu-id="5b776-164">Then, relevant applications and services are configured to submit telemetry data to the gateway instead of a default logging system.</span></span> <span data-ttu-id="5b776-165">网关随后便可以处理数据：聚合、合并或以其他方式设置其格式，然后再将其提交到监视服务以便进行引入和分析。</span><span class="sxs-lookup"><span data-stu-id="5b776-165">The gateway can then process the data: aggregating, combining, or otherwise formatting it before then submitting it to your monitoring service for ingestion and analysis.</span></span>

- <span data-ttu-id="5b776-170">监视系统具有可用于在网关进行处理之后引入日志数据的 API 或其他机制。</span><span class="sxs-lookup"><span data-stu-id="5b776-170">Your monitoring systems have APIs or other mechanisms available to ingest log data after processing by the gateway.</span></span>

<span data-ttu-id="5b776-173">如果对替换难度或成本较高的本地监视系统进行了现有投资，则可能需要来自云工作负荷的遥测数据集成到预先存在的本地监视解决方案中。</span><span class="sxs-lookup"><span data-stu-id="5b776-173">If you have an existing investment in on-premises monitoring systems that would be difficult or costly to replace, you may need to integrate the telemetry from your cloud workloads into preexisting on-premises monitoring solutions.</span></span> <span data-ttu-id="5b776-174">在混合本地监视系统中，本地遥测数据会继续使用现有本地监视系统。</span><span class="sxs-lookup"><span data-stu-id="5b776-174">In a hybrid on-premises monitoring system, on-premises telemetry data continues to use the existing on-premises monitoring system.</span></span> <span data-ttu-id="5b776-175">基于云的遥测数据会直接发送到云监视系统，或者数据会与工作负荷一起存储在云中，随后按固定间隔汇总并引入到本地系统中。</span><span class="sxs-lookup"><span data-stu-id="5b776-175">Cloud-based telemetry data is either sent to the cloud monitoring system directly, or the data is stored on the cloud alongside your workloads and then compiled and ingested into the on-premises system at regular intervals.</span></span>

<span data-ttu-id="5b776-176">**本地混合监视假设**。</span><span class="sxs-lookup"><span data-stu-id="5b776-176">**On-premises hybrid monitoring assumptions**.</span></span> <span data-ttu-id="5b776-177">使用本地日志记录和报告系统进行混合监视假设存在以下情况：</span><span class="sxs-lookup"><span data-stu-id="5b776-177">Using an on-premises logging and reporting system for hybrid monitoring assumes the following:</span></span>

# <a name="real-time-processing"></a><span data-ttu-id="62085-102">实时处理</span><span class="sxs-lookup"><span data-stu-id="62085-102">Real time processing</span></span>

<span data-ttu-id="62085-103">实时处理以最低的延迟对实时捕获的数据流进行处理，以生成实时（或近乎实时）报告或自动响应。</span><span class="sxs-lookup"><span data-stu-id="62085-103">Real time processing deals with streams of data that are captured in real-time and processed with minimal latency to generate real-time (or near-real-time) reports or automated responses.</span></span> <span data-ttu-id="62085-104">例如，实时监视解决方案可以使用传感器数据来检测车流量。</span><span class="sxs-lookup"><span data-stu-id="62085-104">For example, a real-time traffic monitoring solution might use sensor data to detect high traffic volumes.</span></span> <span data-ttu-id="62085-105">可以使用此数据动态更新地图以显示交通堵塞情况，或自动启动堵塞车道或其他交通管理系统。</span><span class="sxs-lookup"><span data-stu-id="62085-105">This data could be used to dynamically update a map to show congestion, or automatically initiate high-occupancy lanes or other traffic management systems.</span></span>

![实时处理管道图](./images/real-time-pipeline.png)

<span data-ttu-id="62085-107">实时处理定义为处理无约束输入数据流，处理时要求延迟极短 &mdash; 以毫秒或秒计。</span><span class="sxs-lookup"><span data-stu-id="62085-107">Real-time processing is defined as the processing of unbounded stream of input data, with very short latency requirements for processing &mdash; measured in milliseconds or seconds.</span></span> <span data-ttu-id="62085-108">此传入数据通常以非结构化或半结构化格式（例如 JSON）传送，其处理要求与[批处理](./batch-processing.md)相同，但周转时间更短，可支持实时使用。</span><span class="sxs-lookup"><span data-stu-id="62085-108">This incoming data typically arrives in an unstructured or semi-structured format, such as JSON, and has the same processing requirements as [batch processing](./batch-processing.md), but with shorter turnaround times to support real-time consumption.</span></span>

<span data-ttu-id="62085-109">处理的数据通常写入已针对分析和可视进行优化的分析数据存储。</span><span class="sxs-lookup"><span data-stu-id="62085-109">Processed data is often written to an analytical data store, which is optimized for analytics and visualization.</span></span> <span data-ttu-id="62085-110">还可以将处理的数据直接引入分析和报告层，以用于分析、商业智能和实时仪表板可视化。</span><span class="sxs-lookup"><span data-stu-id="62085-110">The processed data can also be ingested directly into the analytics and reporting layer for analysis, business intelligence, and real-time dashboard visualization.</span></span>

## <a name="challenges"></a><span data-ttu-id="62085-111">挑战</span><span class="sxs-lookup"><span data-stu-id="62085-111">Challenges</span></span>

<span data-ttu-id="62085-112">实时处理解决方案的最大难题之一是实时引入、处理和存储消息，尤其是消息量很大时。</span><span class="sxs-lookup"><span data-stu-id="62085-112">One of the big challenges of real-time processing solutions is to ingest, process, and store messages in real time, especially at high volumes.</span></span> <span data-ttu-id="62085-113">执行处理时不能阻塞引入管道。</span><span class="sxs-lookup"><span data-stu-id="62085-113">Processing must be done in such a way that it does not block the ingestion pipeline.</span></span> <span data-ttu-id="62085-114">数据存储必须支持大量写入。</span><span class="sxs-lookup"><span data-stu-id="62085-114">The data store must support high-volume writes.</span></span> <span data-ttu-id="62085-115">另一个难题是快速处理数据，例如，实时生成警报，或者在实时（或近乎实时）仪表板中显示数据。</span><span class="sxs-lookup"><span data-stu-id="62085-115">Another challenge is being able to act on the data quickly, such as generating alerts in real time or presenting the data in a real-time (or near-real-time) dashboard.</span></span>

## <a name="architecture"></a><span data-ttu-id="62085-116">体系结构</span><span class="sxs-lookup"><span data-stu-id="62085-116">Architecture</span></span>

<span data-ttu-id="62085-117">实时处理体系结构包含以下逻辑组件。</span><span class="sxs-lookup"><span data-stu-id="62085-117">A real-time processing architecture has the following logical components.</span></span>

- <span data-ttu-id="62085-118">**实时消息引入。**</span><span class="sxs-lookup"><span data-stu-id="62085-118">**Real-time message ingestion.**</span></span> <span data-ttu-id="62085-119">该体系结构必须提供一种方法来捕获和存储流处理使用方使用的实时消息。</span><span class="sxs-lookup"><span data-stu-id="62085-119">The architecture must include a way to capture and store real-time messages to be consumed by a stream processing consumer.</span></span> <span data-ttu-id="62085-120">在简单的场合中，可将此服务实施为简单的数据存储，以便将新消息放入其中的某个文件夹。</span><span class="sxs-lookup"><span data-stu-id="62085-120">In simple cases, this service could be implemented as a simple data store in which new messages are deposited in a folder.</span></span> <span data-ttu-id="62085-121">但是，解决方案通常需要一个充当消息缓冲区的消息中转站，例如 Azure 事件中心。</span><span class="sxs-lookup"><span data-stu-id="62085-121">But often the solution requires a message broker, such as Azure Event Hubs, that acts as a buffer for the messages.</span></span> <span data-ttu-id="62085-122">消息中转站应支持横向扩展处理和可靠传递。</span><span class="sxs-lookup"><span data-stu-id="62085-122">The message broker should support scale-out processing and reliable delivery.</span></span>

- <span data-ttu-id="62085-123">**流处理。**</span><span class="sxs-lookup"><span data-stu-id="62085-123">**Stream processing.**</span></span> <span data-ttu-id="62085-124">捕获实时消息后，解决方案必须通过筛选、聚合以及准备用于分析的数据来处理消息。</span><span class="sxs-lookup"><span data-stu-id="62085-124">After capturing real-time messages, the solution must process them by filtering, aggregating, and otherwise preparing the data for analysis.</span></span>

- <span data-ttu-id="62085-125">**分析数据存储。**</span><span class="sxs-lookup"><span data-stu-id="62085-125">**Analytical data store.**</span></span> <span data-ttu-id="62085-126">许多大数据解决方案会先准备用于分析的数据，然后以结构化格式提供已处理的数据供分析工具查询。</span><span class="sxs-lookup"><span data-stu-id="62085-126">Many big data solutions are designed to prepare data for analysis and then serve the processed data in a structured format that can be queried using analytical tools.</span></span>

- <span data-ttu-id="62085-127">**分析和报告。**</span><span class="sxs-lookup"><span data-stu-id="62085-127">**Analysis and reporting.**</span></span> <span data-ttu-id="62085-128">大多数大数据解决方案的目的是通过分析和报告提供对数据的见解。</span><span class="sxs-lookup"><span data-stu-id="62085-128">The goal of most big data solutions is to provide insights into the data through analysis and reporting.</span></span>

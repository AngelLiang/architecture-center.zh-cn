<span data-ttu-id="cb938-111">API 网关位于客户端与服务之间。</span><span class="sxs-lookup"><span data-stu-id="cb938-111">An API gateway sits between clients and services.</span></span> <span data-ttu-id="cb938-112">它充当反向代理，将来自客户端的请求路由到服务。</span><span class="sxs-lookup"><span data-stu-id="cb938-112">It acts as a reverse proxy, routing requests from clients to services.</span></span> <span data-ttu-id="cb938-113">它还可以执行各种横切任务，例如身份验证、SSL 终止和速率限制。</span><span class="sxs-lookup"><span data-stu-id="cb938-113">It may also perform various cross-cutting tasks such as authentication, SSL termination, and rate limiting.</span></span> <span data-ttu-id="cb938-114">如果未部署网关，则客户端必须直接向前端服务发送请求。</span><span class="sxs-lookup"><span data-stu-id="cb938-114">If you don't deploy a gateway, clients must send requests directly to front-end services.</span></span> <span data-ttu-id="cb938-115">但是，直接向客户端公开服务会造成一些潜在问题：</span><span class="sxs-lookup"><span data-stu-id="cb938-115">However, there are some potential problems with exposing services directly to clients:</span></span>

<span data-ttu-id="cb938-127">网关可以通过将客户端与服务分开，来帮助解决这些问题。</span><span class="sxs-lookup"><span data-stu-id="cb938-127">A gateway helps to address these issues by decoupling clients from services.</span></span> <span data-ttu-id="cb938-128">网关可以执行许多不同的功能，但我们不一定需要所有这些功能。</span><span class="sxs-lookup"><span data-stu-id="cb938-128">Gateways can perform a number of different functions, and you may not need all of them.</span></span> <span data-ttu-id="cb938-129">这些功能可划分到以下设计模式：</span><span class="sxs-lookup"><span data-stu-id="cb938-129">The functions can be grouped into the following design patterns:</span></span>

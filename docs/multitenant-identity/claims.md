## <a name="using-claims-for-authorization"></a><span data-ttu-id="dab76-201">使用声明进行授权</span><span class="sxs-lookup"><span data-stu-id="dab76-201">Using claims for authorization</span></span>

<span data-ttu-id="dab76-202">使用声明时，用户的标识不再是单一实体。</span><span class="sxs-lookup"><span data-stu-id="dab76-202">With claims, a user's identity is no longer a monolithic entity.</span></span> <span data-ttu-id="dab76-203">例如，用户可能具有电子邮件地址、电话号码、生日、性别等信息。所有这些信息可能存储在用户的 IDP 中。</span><span class="sxs-lookup"><span data-stu-id="dab76-203">For example, a user might have an email address, phone number, birthday, gender, etc. Maybe the user's IDP stores all of this information.</span></span> <span data-ttu-id="dab76-204">但是，对用户进行身份验证时，通常会以声明的形式收到其中的一部分信息。</span><span class="sxs-lookup"><span data-stu-id="dab76-204">But when you authenticate the user, you'll typically get a subset of these as claims.</span></span> <span data-ttu-id="dab76-205">在此模型中，用户的标识只是一堆声明。</span><span class="sxs-lookup"><span data-stu-id="dab76-205">In this model, the user's identity is simply a bundle of claims.</span></span> <span data-ttu-id="dab76-206">在针对用户做出授权决策时，会查找特定的声明集。</span><span class="sxs-lookup"><span data-stu-id="dab76-206">When you make authorization decisions about a user, you will look for particular sets of claims.</span></span> <span data-ttu-id="dab76-207">换而言之，问题“用户 X 是否可以执行操作 Y”最终变成了“用户 X 是否具有声明 Z”。</span><span class="sxs-lookup"><span data-stu-id="dab76-207">In other words, the question "Can user X perform action Y" ultimately becomes "Does user X have claim Z".</span></span>

<span data-ttu-id="dab76-208">下面是检查声明的一些基本模式。</span><span class="sxs-lookup"><span data-stu-id="dab76-208">Here are some basic patterns for checking claims.</span></span>

* <span data-ttu-id="dab76-209">若要检查用户是否具有包含特定值的特定声明：</span><span class="sxs-lookup"><span data-stu-id="dab76-209">To check that the user has a particular claim with a particular value:</span></span>

   ```csharp
   if (User.HasClaim(ClaimTypes.Role, "Admin")) { ... }
   ```

   <span data-ttu-id="dab76-210">此代码检查用户是否具有包含值“Admin”的 Role 声明。</span><span class="sxs-lookup"><span data-stu-id="dab76-210">This code checks whether the user has a Role claim with the value "Admin".</span></span> <span data-ttu-id="dab76-211">如果用户没有 Role 声明或者具有多个 Role 声明，此代码不会做出正确的处理。</span><span class="sxs-lookup"><span data-stu-id="dab76-211">It correctly handles the case where the user has no Role claim or multiple Role claims.</span></span>
  
   <span data-ttu-id="dab76-212">**ClaimTypes** 类定义常用声明类型的常量。</span><span class="sxs-lookup"><span data-stu-id="dab76-212">The **ClaimTypes** class defines constants for commonly-used claim types.</span></span> <span data-ttu-id="dab76-213">但是，可对声明类型使用任何字符串值。</span><span class="sxs-lookup"><span data-stu-id="dab76-213">However, you can use any string value for the claim type.</span></span>
* <span data-ttu-id="dab76-214">当你认为某个声明类型最多只有一个值时，若要获取该声明类型的单个值，请使用以下代码：</span><span class="sxs-lookup"><span data-stu-id="dab76-214">To get a single value for a claim type, when you expect there to be at most one value:</span></span>

  ```csharp
  string email = User.FindFirst(ClaimTypes.Email)?.Value;
  ```

* <span data-ttu-id="dab76-215">若要获取某个声明类型的所有值，请使用以下代码：</span><span class="sxs-lookup"><span data-stu-id="dab76-215">To get all the values for a claim type:</span></span>

  ```csharp
  IEnumerable<Claim> groups = User.FindAll("groups");
  ```
